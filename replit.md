# Workspace

## Overview

This project now includes a Streamlit web application for managing a school cash fund, backed by a local SQLite database.

## Stack

- **Primary app**: Python 3.11 + Streamlit
- **Database**: SQLite (`caisse_scolaire.db`, created automatically on first launch)
- **PDF generation**: fpdf2
- **Monorepo tooling**: pnpm workspaces remain available for existing TypeScript packages
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **Existing API framework**: Express 5
- **Existing shared DB tooling**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Streamlit App

- `app.py` — main school cash fund app
- `main.py` — compatibility wrapper that starts `app.py`
- `.streamlit/config.toml` — Streamlit server configuration
- `caisse_scolaire.db` — SQLite database generated by the app and used for persistence
- `logo.png` — school logo used in receipt and monthly summary PDFs
- Access is protected by a simple password gate using `CSD2026`.
- Receipts and monthly summaries can be exported as PDF files.

## Key Commands

- `streamlit run app.py --server.port 5000` — run the school cash fund app
- `pnpm run typecheck` — full typecheck across existing TypeScript packages
- `pnpm run build` — typecheck + build all existing TypeScript packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.
